## Model에서 ToString
1. 이전에는 model에서 toString을 쓰는 것이 나쁘다고 생각
2. 객체에 대한 비지니스 로직이면 해당 객체에서 써야 한다고 생각해서 toString을 구현
3. 하지만 테스트 코드를 작성하면서 너무 폐쇄적이면 테스트하기 어렵
   테스트 하기 좋은 코드가 좋은 코드라고 생각하여 toString을 없앰

## Random 테스트
